Autonomous Agents: Building with Hermes Agent and OpenClaw

Introduction: The Age of Autonomous AI

We're entering an era where AI doesn't just respond to prompts—it takes autonomous action. Tools like Claude Code represent a fundamental shift from "AI as assistant" to "AI as agent." But how do you harness this power to build truly autonomous systems?

In this article, I'll explore how to leverage AI agents for autonomous workflows, focusing on two powerful tools: Hermes Agent by Nous Research—a self-improving AI that builds skills from experience—and OpenClaw—the viral open-source framework that hit 214,000 GitHub stars by turning LLMs into autonomous agents across your existing apps.

The Agentic Loop

Claude Code operates in an agentic loop:

  1. Observe - Read files, check status, gather context
  2. Think - Plan the approach, break down tasks
  3. Act - Execute tools (edit, run, commit)
  4. Verify - Check results, run tests
  5. Iterate - Repeat until goal is achieved

This loop enables complex, multi-step tasks that would be tedious to do manually.

Building Autonomous Systems

Hermes Agent: The Self-Improving AI

Hermes Agent by Nous Research is a fully open-source autonomous agent (MIT license) with a unique approach: it learns and grows with you.

Key Innovation: Skill Documents

Instead of dumping context into a vector database, Hermes builds Skill Documents—searchable markdown files that grow as the agent works. When it solves a problem, it writes a skill document so it never has to solve it again. The longer it runs, the more capable it gets.

  • Cross-session memory - Remembers your projects, preferences, and context forever
  • Multi-platform integration - Telegram, Discord, WhatsApp, Slack, Signal, Home Assistant
  • Multiple backends - Local, Docker, SSH, Singularity, Modal (serverless)
  • Model flexibility - Supports OpenRouter (200+ models), OpenAI, Claude, and custom endpoints
# Install Hermes Agent
pip install hermes-agent

# Initialize with your preferred model
hermes init --model claude-3-opus

# Start the agent
hermes run

# Switch models instantly
hermes model openrouter/anthropic/claude-3.5-sonnet

OpenClaw: The Viral Agent Framework

OpenClaw (formerly Clawdbot) by Peter Steinberger is the open-source phenomenon that hit 214,000 GitHub stars—faster growth than Docker, Kubernetes, or React ever saw.

What makes OpenClaw different: It gives the LLM itself the ability to write, save, and execute code. The LLM becomes the developer.

The Five-Component Architecture:

  1. Gateway - Routes messages from Slack, WhatsApp, Telegram, Discord
  2. Brain - Orchestrates LLM calls using the ReAct reasoning loop
  3. Memory - Stores persistent context in local Markdown files
  4. Skills - Plug-in capabilities for actions (community-contributed)
  5. Heartbeat - Schedules tasks and monitors inboxes
# OpenClaw skill example
from openclaw import Skill, action

class GitHubSkill(Skill):
    """Autonomous GitHub management"""

    @action
    async def create_pr(self, repo: str, title: str, body: str):
        """Create a pull request from current changes"""
        # OpenClaw handles auth, context, and execution
        branch = await self.git.create_branch(f"feature/{slugify(title)}")
        await self.git.commit_all(title)
        await self.git.push(branch)
        return await self.github.create_pr(repo, title, body)

    @action
    async def fix_issue(self, issue_url: str):
        """Autonomously fix a GitHub issue"""
        issue = await self.github.get_issue(issue_url)
        plan = await self.brain.plan(f"Fix: {issue.title}\n{issue.body}")

        for step in plan.steps:
            await self.execute(step)

        await self.create_pr(
            repo=issue.repo,
            title=f"Fix #{issue.number}: {issue.title}",
            body=f"Closes #{issue.number}\n\n{plan.summary}"
        )

Hermes vs OpenClaw: When to Use Each

Hermes Agent excels at personal AI assistance with its self-improving skill system. OpenClaw shines in team environments with its extensive integrations and larger community ecosystem.

  • Self-improving over time - Hermes (built-in skill learning) vs OpenClaw (manual skill creation)
  • Privacy-focused - Both excellent, OpenClaw stores everything in local Markdown
  • Community skills - Hermes has Skills Hub (agentskills.io), OpenClaw has ClawHub (larger ecosystem)
  • Enterprise use - OpenClaw has more integrations out of the box

Building Your Own Autonomous Agent

Key Components

To build an autonomous agent, you need:

  1. Task Definition - Clear goals and success criteria
  2. Context Gathering - Ability to read and understand state
  3. Action Space - Tools the agent can use
  4. Verification - Ways to check if actions succeeded
  5. Error Recovery - Handling failures gracefully

Example: Autonomous PR Agent

# Autonomous PR creation agent
class PRAgent:
    def __init__(self, repo, llm_client):
        self.repo = repo
        self.llm = llm_client

    async def create_pr(self, issue):
        # 1. Understand the issue
        context = await self.gather_context(issue)

        # 2. Plan the implementation
        plan = await self.llm.plan(issue, context)

        # 3. Implement changes
        for task in plan.tasks:
            await self.implement(task)

        # 4. Verify changes
        test_results = await self.run_tests()
        if not test_results.passed:
            await self.fix_tests(test_results)

        # 5. Create PR
        pr = await self.repo.create_pr(
            title=plan.title,
            body=self.generate_description(plan, context),
            branch=self.branch_name
        )

        return pr

Best Practices for Autonomous Agents

Safety First

  • Sandboxing - Limit agent actions to safe boundaries
  • Review gates - Require human approval for critical actions
  • Rollback capability - Always be able to undo agent actions
  • Logging - Track everything for debugging and auditing

Effective Prompting for Agents

  • Be specific - Clear goals lead to better results
  • Provide context - Help agents understand the bigger picture
  • Define constraints - Set boundaries for acceptable solutions
  • Specify verification - Tell agents how to check their work
# Good agent prompt
"Implement a rate limiter for the API endpoints.
Requirements:
- Use Redis for distributed rate limiting
- 100 requests per minute per user
- Return 429 status when exceeded
Verify by: Running the rate_limit_test.py suite"

# Less effective prompt
"Add rate limiting"

Incremental Autonomy

Start with supervised automation and gradually increase autonomy:

  1. Assisted - Agent suggests, human executes
  2. Supervised - Agent executes, human approves
  3. Autonomous - Agent executes within defined boundaries
  4. Self-improving - Agent optimizes its own behavior

The Future: Self-Orchestrating Systems

The ultimate vision is systems where:

  • Agents spawn other agents as needed
  • Work is dynamically distributed based on complexity
  • Systems self-heal and self-optimize
  • Human oversight is strategic, not tactical

We're not quite there yet, but tools like Claude Code are the building blocks.
